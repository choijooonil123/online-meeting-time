# 원격수업 학생 상담관리 시스템

Firebase Hosting + Firestore + Cloud Functions 기반 전화면담 예약 시스템입니다.

## 포함 기능

- 지도교수 상담 가능 시간을 등록하면 10분 단위 슬롯으로 자동 분할
- 학생 예약 신청 후 `승인대기`
- 지도교수 승인 시 `면담확정`, 반려 시 `반려/취소`
- 학생용 `내 예약 확인` 화면
- 예약 신청 후 학생 학번이 포함된 확인 링크 제공
- 교수용 승인 대기 목록과 월간 확정 일정 캘린더
- 교수용 학생 명단 저장, 회차별 미신청 학생 자동 산출, 카카오톡용 독려 메시지 복사
- 승인 완료 시 학생과 교수에게 문자 발송하는 Twilio 연동 코드

## 디렉터리 구조

- `public/`: Firebase Hosting 정적 웹앱
- `functions/`: 예약 처리, 승인, 문자 발송 Cloud Functions
- `firestore.rules`: 공개 데이터와 비공개 예약 데이터 접근 제어

## 배포 전 준비

1. Firebase 프로젝트 `online-meeting-time`에 Hosting, Firestore, Functions를 활성화합니다.
2. Node.js 20과 Firebase CLI를 준비합니다.
3. `functions` 폴더에서 의존성을 설치합니다.

```bash
cd functions
npm install
```

4. 문자 발송용 Twilio 런타임 설정을 입력합니다.

```bash
firebase functions:config:set twilio.account_sid="ACxxxx" twilio.auth_token="xxxx" twilio.phone_number="+82xxxxxxxxx"
```

5. 프로젝트 루트에서 배포합니다.

```bash
firebase deploy
```

## 초기 관리자 코드

- 기본 관리자 코드는 `1234`입니다.
- 첫 로그인 후 `functions/index.js` 또는 별도 UI 확장으로 변경하는 것을 권장합니다.

## 학생 예약 확인 링크

예약 신청 성공 시 다음 형식으로 표시됩니다.

```text
https://online-meeting-time.web.app/?studentNo=20260001
```

해당 링크는 학번만 미리 채워주며, 학생은 이름과 휴대폰 뒷자리를 추가로 입력해 상태를 확인합니다.

## 미신청 학생 관리

- 교수 화면에서 학생 명단을 `학번 이름` 형식으로 한 줄씩 붙여넣어 저장합니다.
- 학기 설정에서 `1차 집중면담 시작/종료일`, `2차 집중면담 시작/종료일`을 설정할 수 있으며 각 기간은 14일 이내로 운영합니다.
- 상담 가능 시간 슬롯은 해당 회차의 집중면담기간 안에서만 생성됩니다.
- 학생은 공개된 예약 시간이 모두 어려운 경우 `대체 면담 요청`으로 가능한 시간대와 사유를 남길 수 있습니다.
- 학생은 정규 면담과 별도로 `수시면담 신청`을 남겨 원하는 시간대와 상담 주제를 직접 요청할 수 있습니다.
- 교수 화면에서는 대체 면담 요청 목록을 보고 전화 연결 후 `조율 완료` 또는 `취소`로 처리할 수 있습니다.
- 교수 화면에서는 수시면담 신청도 별도 목록으로 확인하고 전화 연결 후 `조율 완료` 또는 `취소`로 처리할 수 있습니다.
- 회차별로 `승인대기` 또는 `면담확정` 상태가 없는 학생을 자동으로 `미신청 학생`으로 계산합니다.
- `전체 독려 메시지 복사`는 단톡/오픈채팅용 안내문을, `개인별 메시지 복사`는 학생별 안내문을 클립보드에 복사합니다.
- 교수 화면에는 학생/교수 전화번호가 일반 텍스트로 표시됩니다.
- 확정된 면담에서 통화가 되지 않으면 `통화불가 처리`를 눌러 해당 학생을 다시 `미신청`으로 분류할 수 있습니다.
